Configure Lifecycle Hooks

Lifecycle hooks are events broadcast into the container before, during, or after specific events take place. These hooks are synchronous per pod and will block any other actions from taking place, including liveness probes.

This hook is sent immediately after a container is created. It notifies the container that it has been created.

No parameters are passed by this hook.

This hook is called immediately before a container is terminated. Since it blocks, this is useful for saving ephemeral data or safely shutting down the application.

A single parameter, reason is passed to the handler which contains the reason for termination.
